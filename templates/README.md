| `memory/mistakes-a4.html` | 錯題本（錯因分類、+3/+7/+15 天重做排程） | A4 直式，一頁兩題 |

flashcards 雙面列印要選「**沿長邊翻轉**」，裁切前先對光確認正背編號重疊。

## 用法一：手動複製修改

1. 複製要用的 template，改檔名。
2. 每個檔案開頭的 `<!-- ... -->` 註解說明了哪些區塊可以增刪複製。
3. 只改內容，**不要動 `<head>` 裡的字型設定**——那是列印中文的關鍵。

## 用法二：叫 AI 產出

直接對 Claude 說，例如：

> 用 templates/report-a4.html 的格式，幫我產一份「××系統導入評估報告」，內容是……

> 照 templates/slides-16x9.html 幫我做一份 8 頁的提案簡報，主題是……

每個 template 開頭的註解已寫好給 AI 的產出規則（保留字型設定、
沿用既有版型、不加 emoji），AI 讀檔後會照規則產出。

## 列印成 PDF（Cmd+P）

1. 用 Chrome 開啟 HTML，按 Cmd+P。
2. 目的地選「另存為 PDF」。
3. 展開「更多設定」：
   - **勾選「背景圖形」**（否則深色投影片、表格底色印不出來）
   - 邊界選「**預設**」即可（紙張邊界已寫在檔案的 `@page` 裡）；若出現多餘白邊改選「無」
   - 取消「頁首及頁尾」
4. 簡報檔不必手動選橫向，`@page` 已指定 16:9 紙張。

## 交付前驗證（建議）

別只看螢幕，先用 headless Chrome 印一份確認中文都在：

```bash
"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" \
  --headless=new --disable-gpu --no-pdf-header-footer \
  --print-to-pdf="/tmp/check.pdf" "file:///絕對路徑/檔案.html"
open /tmp/check.pdf
```

確認：中文完整、文字可選取、分頁正確。

## 新增 template 的鐵則

macOS 上 Chrome 列印 PDF 時**無法嵌入系統中文字型 PingFang**：
螢幕顯示正常，但 PDF 裡的中文會整個消失（詳見上層目錄
`PDF匯出中文消失-問題說明.md`）。因此：

1. `<head>` 一律載入 Google Fonts 的 Noto Sans TC：

   ```html
   <link rel="preconnect" href="https://fonts.googleapis.com">
   <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
   <link href="https://fonts.googleapis.com/css2?family=Noto+Sans+TC:wght@400;500;700&display=swap" rel="stylesheet">
   ```

2. 字型鏈一律明確指定可嵌入的中文 fallback，**禁止**讓中文落到
   `system-ui`、`-apple-system` 或裸 `sans-serif`：

   ```css
   font-family: 'Noto Sans TC', 'Heiti TC', 'Arial Unicode MS', 'Microsoft JhengHei', sans-serif;
   ```

3. 等寬字型（程式碼區塊）也要補中文 fallback：

   ```css
   font-family: 'SF Mono', 'Consolas', 'Noto Sans TC', 'Heiti TC', monospace;
   ```

4. 首次開啟需要網路（抓 Google Fonts）；離線時會落到 Heiti TC，
   仍可正常列印，只是字重層次略少。
