會員系統重構暨資料遷移服務提案

提案編號P-2026-031 提案對象新銳製造股份有限公司 提案日期2026-06-11 有效期限2026-07-31

1. 背景與目標

貴公司現行會員系統建置於 2018 年，隨會員數成長至 42 萬，已出現批次作業逾時、促銷期間查詢延遲、新功能開發周期過長等問題。本提案目標為：在不中斷既有服務的前提下，於六個月內完成系統重構與全量資料遷移，並將促銷高峰的查詢 P95 延遲控制在 300 毫秒以內。

提案重點 採「雙軌並行、分批切換」策略：新舊系統並行六週，依會員分群分四批切換，任一批次出現異常可於 30 分鐘內回切，將營運風險降至最低。

2. 方案說明

2.1 系統架構

新系統採服務化架構，將會員資料、權益計算、通知發送拆分為獨立服務，以 API 閘道統一對外。資料庫由單體 MySQL 遷移至 PostgreSQL，並導入讀寫分離與快取層。

2.2 資料遷移

遷移採「全量初始 ＋ 異動同步」模式：先完成歷史資料全量搬遷與比對，再以異動日誌持續同步至切換日。每批切換前執行三輪資料一致性核驗，比對結果作為切換放行條件。

3. 工作範圍與交付物

# 工作項目 內容說明 交付物與驗收方式
1 需求確認與系統設計 既有功能盤點、新架構設計、資料模型設計 系統設計文件；經貴我雙方設計審查會議通過
2 核心服務開發 會員、權益、通知三項服務與 API 閘道 通過全部驗收測試案例（雙方共同擬定，約 120 條）
3 資料遷移 全量搬遷、異動同步、四批分群切換 每批一致性核驗報告，差異筆數為零
4 上線與保固 切換執行、監控建置、上線後三個月保固 監控儀表板；保固期內缺陷修復回應時間 4 小時內

不含項目：行動 App 改版、客服系統整合、歷史訂單系統之改造。如有需要另案報價。
