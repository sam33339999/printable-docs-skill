# 問題（考理解，不考填空） 答案要點 自測記錄
1 TCP 建立連線為什麼需要三次握手，兩次不行的原因是什麼？ 雙方都要確認「我能送、對方能收」；兩次無法確認 client 的接收能力，且舊的重複 SYN 會被誤建連線。 R1　R2　R3
2 慢啟動和壅塞避免的差別是什麼？什麼事件會讓 TCP 從一個切到另一個？ 慢啟動指數成長、壅塞避免線性成長；cwnd 達 ssthresh 時切換，逾時則 ssthresh 減半並回到慢啟動。 R1　R2　R3
3 UDP 沒有可靠傳輸，為什麼 DNS、視訊串流反而選它？ 無握手延遲、無重傳隊頭阻塞、標頭小；應用層可自行決定要不要補可靠性（如 QUIC）。 R1　R2　R3
4 TIME_WAIT 狀態存在的目的是什麼？等多久、為什麼是這個長度？ 確保最後的 ACK 送達、讓舊封包在網路中消亡；等 2MSL，涵蓋一去一回的最大存活時間。 R1　R2　R3
5 流量控制和壅塞控制都在「踩煞車」，控制的對象有什麼不同？ 流量控制保護「接收端」（rwnd，接收緩衝區）；壅塞控制保護「網路」（cwnd）；實際視窗取兩者較小值。 R1　R2　R3
6 收到三個重複 ACK 時 TCP 做什麼？為什麼處理方式比逾時溫和？ 快速重傳＋快速恢復（cwnd 減半）；重複 ACK 代表後續封包仍在到達，網路沒有全面壅塞。 R1　R2　R3
